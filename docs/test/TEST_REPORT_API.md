## 📝 测试发现

### ✅ 正常工作的 API

1. **文件列表** - `GET /files` ✅
2. **创建目录** - `POST /files/add` ✅
3. **重命名** - `POST /files/edit` (参数：`fid`, `file_name`) ✅
4. **用户设置** - `GET /user/setting` ✅
5. **Cookie 认证** - 完整 Cookie 字符串有效 ✅

### ❌ 失败的 API

1. **删除文件** - `POST /files/delete` ❌
   - 错误：`服务器开小差了，稍后再试吧`
   - 可能原因：服务器临时问题/需要签名参数

### ⚠️ 发现的问题

1. **Cookie 有效期短** - 浏览器复制的 Cookie 几小时后失效
2. **删除 API 不稳定** - 可能受服务器限流影响
3. **API 参数格式** - 需要使用 `fid` 而非 `cid`

### 🔧 解决方案

1. **Cookie 获取** - 使用浏览器插件实时获取（参考 `yenkn/115-to-alist`）
2. **扫码登录** - 使用官方 API 获取长期 Cookie（参考 `ChenyangGao/qrcode_cookie_115`）
3. **API 参数** - 使用项目中封装的 `FileOperations` 类

## 📝 测试发现

### ✅ 正常工作的 API

1. **文件列表** - `GET /files` ✅
2. **创建目录** - `POST /files/add` ✅
3. **Cookie 认证** - 完整 Cookie 字符串有效 ✅

### ⚠️ 发现的问题

1. **Cookie 格式要求** - 需要完整的 Cookie 字符串，包含所有字段
2. **POST 参数格式** - 需要使用 `application/x-www-form-urlencoded`
3. **部分 API 端点** - `/user/mine` 返回 404（可能已废弃）

### 🔧 解决方案

1. Cookie 存储添加 `_raw` 字段保存完整字符串
2. HTTP Client 需要正确设置 Content-Type
3. 使用有效的 API 端点
