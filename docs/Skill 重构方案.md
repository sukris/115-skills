# 115 Skills 重构方案 - 聊天场景优化

> **分析时间：** 2026-03-15 17:35 GMT+8  
> **目的：** 基于新 API 发现，优化 Skill 设计，更贴合聊天场景

---

## 📊 当前设计分析

### 现有 Skill 功能
```
1. 登录认证 🔐
2. 文件浏览 📁
3. 文件搜索 🔍
4. 分享转存 🔄
5. 离线下载 ⬇️
6. 智能整理 🤖
7. 容量查询 📊
8. 清理建议 💡
```

### 问题点
| 问题 | 说明 | 影响 |
|------|------|------|
| ❌ 功能颗粒度大 | 一个 Skill 包含太多功能 | 用户不知道能说什么 |
| ❌ 缺少上下文 | 每次都是独立请求 | 无法连续对话 |
| ❌ 缺少主动推荐 | 用户需要知道具体命令 | 学习成本高 |
| ❌ 缺少状态管理 | 不知道当前在做什么 | 体验割裂 |
| ❌ 缺少快捷操作 | 需要输入完整命令 | 不够便捷 |

---

## 🎯 优化方案

### 方案 A：场景化 Skill 拆分（推荐）

将单一 Skill 拆分为多个场景化 Skill：

```
115-login          # 登录认证
115-files          # 文件管理
115-share          # 分享转存
115-download       # 离线下载
115-organize       # 智能整理
115-status         # 状态查询
```

**优点：**
- ✅ 每个 Skill 职责清晰
- ✅ 用户可以精准调用
- ✅ 易于维护和扩展

**缺点：**
- ⚠️ 用户需要记住多个 Skill 名称
- ⚠️ 跨场景操作复杂

---

### 方案 B：对话式交互优化（推荐）

保持单一 Skill，但优化交互方式：

#### 1. 上下文感知

```
用户：查看文件
助手：📁 根目录 (156 个文件)
      📄 电影.mp4 (2.1 GB)
      📁 工作文档
      回复"打开工作文档"继续浏览

用户：打开工作文档
助手：📁 工作文档 (23 个文件)
      📄 2026 工作报告.docx
      📄 项目计划.xlsx
```

#### 2. 快捷命令

```
用户：/115 电影
助手：🔍 搜索结果 (8 个)
      📄 电影 1.mp4
      📄 电影 2.mkv
      [1] 下载 [2] 转存 [3] 分享 [4] 删除

用户：1
助手：✅ 已开始下载
```

#### 3. 主动推荐

```
用户：容量
助手：📊 存储空间
      已用：15.84 TB / 58.09 TB
      剩余：42.25 TB (72.4%)
      
      💡 建议：
      - 发现 3 个重复文件 (5.2 GB)
      - 回收站有 12 个文件 (1.8 GB)
      回复"清理"开始优化
```

**新场景：**
